Uma associação define um relacionamento entre duas entidades conceituais. Ela define responsabilidades entre duas classes.

Uma agregação é um caso especial de associação que modela um relacionamento tem um (ou parte de, no jargão da UML) entre pares. Esta agregação é composta (outrora conhecida apenas como "composição" em versões anteriores da UML) quando os objetos participantes dependem uns dos outros para existir.

Um objeto pode ser composto de um ou múltiplos objetos. Se um objeto A é composto de um objeto B, ele depende do objeto B. Isso não significa que o objeto A é composto apenas do objeto B. Ele pode possuir outros objetos em sua composição. Se o relacionamento entre o objeto A e o objeto B for perdido, o significado lógico dos objetos pode ser perdido ou alterado significativamente.
