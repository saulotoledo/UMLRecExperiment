Uma interface é a definição de um tipo, sem especificação de comportamento. Ela contém um conjunto de recursos que permitem a interação entre a unidade e o que lhe é exterior. Estes recursos são as operações, em termos de UML.

Interfaces, quando bem utilizadas, contribuem fortemente para a redução de acoplamento no sistema. Geralmente, elas são representadas por um retângulo com "«interface»", como na imagem a seguir:

Como interfaces são definições, elas não são instanciáveis. Portanto, uma especificação de interface é implementada por um outro elemento. Assim, este outro elemento apresenta uma fachada pública que obedece à especificação da interface.

Classes podem implementar mais de uma interface, e interfaces podem ser implementadas por várias classes. Uma associação entre uma interface e uma classe implica que uma associação de conformidade deve existir entre qualquer implementação daquela interface.
